# Tracker v2.5 — Debt cleanup & hardening

- **Date:** 2026-06-15
- **Status:** Approved, ready for plan
- **Builds on:** `2026-06-15-tracker-ui-v2.4-design.md`

A maintenance round from a codebase audit. No user-facing behavior changes to the
tree/dashboard/status-bar/status logic — this is docs, de-duplication, type
tightening, log-growth bounding, and documenting two accepted approximations.

## Scope (audit clusters)

### D1+D3 — docs & version
1. **Rewrite `README.md`** to reflect v2.4 reality:
   - Two stacked views: `Tasks & Subagents` tree + `Dashboard` webview.
   - Workspace grouping + worktree nesting (`repo ▸ worktree ▸ feature`).
   - Statuses: `active` / `done` / `idle` / `ended`, with what each means.
   - Dismiss (right-click / hover ✕), `Tracker: Clear inactive`, `Tracker: Reset dismissed`, persisted in globalState.
   - Auto-hide of done/ended after `claudeTaskTracker.hideDoneAfterMinutes` (default 30).
   - Live plan detection (plan-file writes), and the new `claudeTaskTracker.logRetentionDays` setting.
   - Full hook list: `SessionStart`, `PostToolUse`(TodoWrite|Write|Edit|MultiEdit), `PreToolUse`(Task), `SubagentStop`, `Stop`, `SessionEnd`.
2. **Add `CHANGELOG.md`** summarizing v2.0 → v2.4 (one bullet group per version) and a v2.5 entry.
3. **Bump `package.json` `version` 0.0.1 → 0.2.0.**

### D2+D4 — code cleanliness
4. **Extract shared helpers** `basename(p)` and `shortId(session)` into new `src/util.ts`; `reducer.ts` and `viewModel.ts` import them (remove both local copies). Identical behavior.
5. **Tighten hook payload types** in `src/hook/core.ts`: replace `tool_input?: any` with a typed `HookToolInput` interface and the `td: any` map callback with a typed item. No `any`.
   ```ts
   export interface HookToolInput {
     todos?: Array<{ content?: string; text?: string; status?: string }>;
     file_path?: string;
     subagent_type?: string;
     description?: string;
   }
   // HookPayload.tool_input?: HookToolInput
   ```
   Narrowing logic unchanged (still guards `Array.isArray(tool_input?.todos)` etc.).

### O1+O2 — bound log growth
6. New setting **`claudeTaskTracker.logRetentionDays`** (number, default `14`, min `0`; `0` = keep everything).
7. New pure module **`src/logRetention.ts`**:
   ```ts
   export function compactEvents(events: TrackerEvent[], now: number, retentionDays: number): TrackerEvent[]
   ```
   - `retentionDays <= 0` → return events unchanged.
   - else → keep events with `ts >= now - retentionDays*86_400_000`.
8. **`store.start()` compacts once at startup**: read events, run `compactEvents`; if it removed any, rewrite the log atomically (write temp file, rename over the original) before the initial `reduce`. `reduce` then reads a bounded file. The per-change `recompute` stays a full read+reduce (acceptable now the file is bounded — no incremental-reduce machinery).
   - `TrackerStore` gains a retention input. Its constructor currently takes only `logPath`; add a second arg `retentionDays: () => number` (a getter so it reads the live setting). `extension.ts` passes `() => getConfiguration('claudeTaskTracker').get('logRetentionDays', 14)`.
   - **Concurrency note:** the log is shared across windows (hooks append from every session). Compaction runs only at startup and only when over the cutoff; temp-write+rename minimizes the window, but a concurrent hook append during the rename could be lost (worst case: one dropped event, rare). Documented, accepted for a local dev tool.

### Q1+Q2 — accepted approximations (document, no behavior change)
9. **Subagent convergence (FIFO):** `SubagentStop` provides no subagent id, so exact attribution is impossible; FIFO (oldest running converges first) is the best available. Keep it; document in README + a code comment. (`buildEvents` already has no id to capture; leave as-is.)
10. **Plan selection (newest-by-mtime):** with v2.4 live plan detection, a freshly-written plan becomes newest and is auto-selected, so newest-by-mtime is correct in practice. Keep it; document in README + a code comment near `findNewestPlan`.

## Testing

- `util`: `basename` (slashes, trailing, empty), `shortId` (uuid segment, no-hyphen).
- `logRetention`: `retentionDays=0` → unchanged; old events dropped, recent kept; boundary (exactly cutoff kept).
- reducer/viewModel/hookCore: unchanged assertions still pass after the import move + type tightening.
- store: a `store.test.ts` case writing a temp log with a very-old event (`ts: 0`) plus a fresh one (`ts: Date.now()`), `retentionDays` getter returning 14 → after `start()` the on-disk file no longer contains the old event and `state` reflects only the fresh one. (Uses real `Date.now()`; `ts:0` is unambiguously past the 14-day cutoff — no clock injection needed.)

## Non-goals

- No incremental/streaming reduce (file is bounded after compaction — not worth the complexity).
- No watcher debounce (O3) or timer-skip (O4) — deferred, low value.
- No change to subagent-attribution or plan-selection *behavior* (Q1/Q2 are documentation-only).
